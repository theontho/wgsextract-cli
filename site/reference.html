Core glossary

Short definitions for terms you will see across WGS Extract, samtools/bcftools, and variant annotation tools.

Alignment

Mapping sequencing reads to a reference genome, producing a BAM or CRAM.

Allele

A version of a sequence at a genomic position. Humans usually have two alleles for autosomal positions.

Annotation

Metadata added to variants, such as gene name, population frequency, predicted effect, or known clinical significance.

Build

The reference genome version used for coordinates, such as hg19, hg38, or T2T.

Coverage

How many reads support a base or region. Average coverage and breadth of coverage are different measures.

Haplogroup

A lineage classification based on Y-DNA or mtDNA markers.

Liftover

Coordinate conversion between reference builds, usually with chain files.

Ploidy

How many copies of a chromosome or region are expected, such as two autosomal copies or one Y chromosome in many male samples.

Variant caller

A tool that reads alignments and emits VCF records for likely variants.

External tool reference

WGS Extract orchestrates established tools instead of reimplementing every algorithm.

samtools / htslib

Alignment viewing, sorting, indexing, conversion, and many low-level operations around BAM/CRAM/VCF files.

bcftools

Variant calling, filtering, annotation, indexing, and VCF/BCF manipulation.

bwa / minimap2

Read alignment for short reads and long reads respectively.

fastqc / fastp

Raw read quality reports, adapter trimming, filtering, and preprocessing.

GATK / DeepVariant

High-accuracy variant calling workflows with different runtime and model tradeoffs.

VEP

Variant Effect Predictor annotations: consequences, genes, transcripts, impact, and plugin/cache-driven metadata.

Privacy and data handling

Whole genome data is uniquely identifying. Treat it as sensitive personal and family information.

Prefer local processing

WGS Extract is designed for local workflows. Be intentional before uploading BAM, CRAM, VCF, FASTQ, or microarray files to third-party sites.

Keep raw data backed up

Preserve original FASTQ/BAM/CRAM files, reference build notes, and tool commands. Derived files can often be regenerated.

Share minimized outputs

When debugging, use fake data, region-limited extracts, or redacted logs instead of whole-genome files whenever possible.

Not medical advice: Variant interpretation can be difficult and clinically consequential. Confirm medically relevant findings through qualified professionals and validated labs.

Troubleshooting checklist

Most failures come from missing tools, mismatched references, absent indexes, unsorted files, permissions, or jobs that need more disk/RAM/time.

Command not found

Run through Pixi: pixi run wgsextract .... Check pixi install completed and pixi run wgsextract deps check passes.

Reference errors

Verify the FASTA exists, is indexed, and matches the build used by the input BAM/CRAM or VCF.

Region not found

Check chromosome naming. Some files use chrM, chrY, chr1; others use MT, Y, 1.

Variant caller fails

Confirm the BAM/CRAM is coordinate sorted, indexed, readable, and aligned to the same reference passed with --ref.

Job is too slow

Run a small --region first, then increase threads and output disk. Whole-genome callers can take hours.

Windows tool issues

Use WSL2 for the broadest compatibility, or follow the native pacman runtime guide.
