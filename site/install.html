Setup guide

Install once, then run repeatable genome workflows.

WGS Extract CLI uses Pixi to manage Python, the app itself, and many external bioinformatics tools. That keeps commands consistent across machines and avoids hand-installing samtools, bcftools, bwa, minimap2, fastp, fastqc, and related tooling one by one.

QuickstartPlatformsReferencesConfigurationVerify

Install Pixi

Install Pixi first, then restart your terminal so the pixi command is on your PATH.

Clone the project

Clone the repository and run pixi install from the project root.

Bootstrap references

Download the metadata and reference-library structure WGS Extract uses to find FASTA, VCF, ploidy, liftover, and annotation assets.

Run a small test

Use info and region-limited commands first. Whole-genome jobs can take hours and hundreds of gigabytes of working space.

macos-linux-wsl2.shrecommended path
# Install Pixi
curl -fsSL https://pixi.sh/install.sh | bash

# Clone and install
git clone https://github.com/theontho/wgsextract-cli.git
cd wgsextract-cli
pixi install

# Initialize reference library data
pixi run wgsextract ref bootstrap
pixi run wgsextract ref library --list
pixi run wgsextract ref library --install hs38

# Verify the app and environment
pixi run wgsextract info --detailed
pixi run wgsextract deps check

Platform notes

Bioinformatics tools are most mature on Unix-like environments. Windows is supported best through WSL2, with native Windows workflows available for some toolchains.

macOS

Supported on Intel and Apple Silicon. Pixi resolves Python and native command-line tools through conda-forge and bioconda where available.

osx-arm64osx-64

Linux and WSL2

The smoothest route for full tool coverage, especially for large WGS workflows, VEP, DeepVariant, PacBio environments, and server-style runs.

Native Windows

Use the Windows PowerShell Pixi installer for Python/app setup. For native bio tools, use the MSYS2 UCRT64 pacman runtime described in the Windows docs.

Reference library setup

Most genome operations need a reference genome and companion files. The reference library lets WGS Extract resolve those assets instead of asking you for every path on every command.

Bootstrap data

ref bootstrap initializes the library scaffolding and common metadata. Then use the library commands to list and install supported builds such as hs38.

pixi run wgsextract ref bootstrap
pixi run wgsextract ref library --list
pixi run wgsextract ref library --install hs38

Index and verify

FASTA references need indexes and dictionaries before alignment and variant callers can use them efficiently.

pixi run wgsextract ref index --ref /path/to/hs38.fa
pixi run wgsextract ref verify --ref /path/to/hs38.fa
Reference builds matter. hg19/GRCh37, hg38/GRCh38, and T2T coordinates differ. Do not mix BAMs, VCFs, annotation files, and gene coordinates from different builds unless you intentionally lift over or re-run the analysis.

Configuration

Use wgsextract config to view the active config path and defaults. You can set input, output, reference, genome library, thread, memory, and external tool paths once instead of repeating them.

Config locations

macOS: ~/.config/wgsextract/config.toml or Application Support. Linux: ~/.config/wgsextract/config.toml. Windows: under AppData.

Genome library

Set genome_library to a folder with one subfolder per person or sample. Then --genome sample-id can resolve inputs and outputs from that folder.

config.tomlexample
input = "/data/genomes/joe/joe.cram"
outdir = "/data/genomes/joe/out"
ref = "/data/reference/hs38.fa"
genome_library = "/data/genome-library"
threads = 8
memory = "16G"
haplogrep_path = "/usr/local/bin/haplogrep"

Verify before running full genomes

Start with environment checks and tiny regions. It is much cheaper to catch missing references, unsorted BAMs, or command syntax errors on chrM than halfway through a full-genome job.

verification-recipes.shsafe first commands
pixi run wgsextract deps check
pixi run wgsextract --input sample.cram info --detailed
pixi run wgsextract --input sample.bam extract mito-vcf --region chrM
pixi run wgsextract --input sample.bam microarray --formats 23andme_v5 --region chrM
